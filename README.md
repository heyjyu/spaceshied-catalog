# 스페이스쉴드 워치 스트랩 카탈로그

기존 운영 구글시트(기종별 통합 List)를 읽어, 워치 스트랩을 **검색·필터**하고
**클릭하면 큰 이미지 + 속성값**을 보여주는 내부 관리용 웹앱.

- 검색: 재질·제품명·스트랩 규격·체결 형태·색상·기종 전체검색 + 드롭다운 필터
- 상세: 표/카드 클릭 → 큰 이미지 + 모든 속성
- 데이터: 시트에서 자동 추출 (이미지 포함) → 정적 파일로 서빙 (백엔드 0)

## 동작 방식 (중요)

운영 시트가 **기종별 23개 탭 + 제목줄/병합셀/운영컬럼**이 섞인 형태라
앱이 시트를 직접 실시간으로 읽지 않고, **변환 파이프라인**으로 깨끗하게 뽑아 씁니다:

```
구글시트 ──fetch.py──> raw/*.csv (탭별) + sheet.xlsx(삽입이미지)
          ──normalize.py──> catalog.csv  +  images/*.jpg
                                    └─> 웹앱(index.html)이 표시
```

- `fetch.py` : 23개 탭 CSV + 시트 전체 xlsx(삽입 이미지 포함) 다운로드
- `normalize.py` : 탭마다 다른 컬럼을 키워드 매핑 → 통일,
  재질/규격 정규화, 체결 형태는 제품명에서 추출, 기종은 탭 이름에서,
  xlsx 삽입 이미지를 (탭·행) 매칭해 `images/`에 리사이즈 저장

### 데이터 갱신 (한 방)

시트를 수정한 뒤 최신 반영:
```bash
./build.sh                 # = fetch.py + normalize.py
# 웹앱 새로고침하면 끝
```

## 지금 바로 보기

```bash
python3 -m http.server 8000
# http://localhost:8000
```
`config.js`의 `SHEET_ID="DEMO"` → 변환된 `sample-data.csv`(= 최신 catalog.csv) 표시.

## 현재 데이터 채움률 (원본 시트 기준)

| 속성 | 채움률 | 비고 |
|---|---|---|
| 제품명 / 기종 | 100% | |
| 재질 | 84% | 35종→실리콘/패브릭/메탈/가죽/스틸 등으로 정규화 |
| 사이즈(mm) | 63% | 규격셀+제품명에서 폭(mm)만 추출 → 잡값 없이 14/18/20/22/24mm 등 8종으로 정리 |
| 체결 형태 | 47% | 체결방식 컬럼 없으면 제품명에서 추출 |
| 이미지 | 52% | 시트에 삽입된 사진 자동 추출(나머지는 시트에 없음) |
| **색상** | **27%** | 원본은 18%였으나, 복합 색상명을 표준색 16버킷으로 정규화해 필터 가능한 행 27%로 ↑ (나머지는 스마트스토어 옵션에만 존재) |

### 남은 일
- **색상**: 채워진 값은 표준색 버킷으로 정규화해 필터 가능(27%). 더 끌어올리려면 (a) 셀러가 `컬러옵션명` 칸을 채우거나 (b) 스마트스토어 옵션을 시트로 가져와야 함
- **수동 처리 탭**: SET·커넥터·날개형·원클릭·가민 D2.3 등은 구조가 달라 자동변환 제외(`normalize.py`의 `SKIP_TABS`)
- 재질 희귀값(강화유리/PMMA/우레탄 등) 일부는 그대로 둠

## 신규 스트랩 등록 (웹 폼)

`add.html`(사이트의 "➕ 신규 등록" 버튼) 에서 신상을 입력하면 시트 "신규입력" 탭에 쌓입니다.
정적 사이트라 시트에 쓰려면 **Google Apps Script** 다리가 필요합니다(무료, 1회 설정):

1. 등록 시트 → 확장프로그램 → Apps Script → `apps-script.gs` 내용 붙여넣기
2. 배포 → 새 배포 → 웹 앱 → 액세스 "모든 사용자" → 웹앱 URL 복사
3. `add.html` 상단 `GAS_URL` 에 그 URL 붙여넣기 → 커밋/푸시
4. 입력한 신상은 시트에 저장 → `./build.sh` 하면 사이트에 반영
   (샘플링날짜·컨텐츠제작자·쿠팡/네이버 등록·이미지 속성 자동 포함)

> 이미지는 Drive "카탈로그_이미지" 폴더에 저장되고 썸네일 URL로 표시됩니다.

## 🔁 다른 셀러에 붙이기 (재사용 템플릿)

이 앱은 **`config.js` 한 파일만** 바꾸면 다른 셀러 시트로 그대로 동작합니다.
HTML/JS/CSS는 손댈 필요 없습니다. (app.js가 헤더 이름을 보고 컬럼·필터를 자동 구성)

### 1) 데이터 연결 (`config.js` 상단)
```js
TITLE:     "○○ 상품 카탈로그",   // 사이트 제목
SHEET_ID:  "1AbC...";            // 구글시트 ID ("DEMO"면 sample-data.csv)
SHEET_GID: "0",                  // 읽을 탭 #gid= 숫자 (또는)
SHEET_NAME:"카탈로그",            // 탭 이름
HEADER_ROW: 1,                   // 헤더(컬럼명)가 몇 번째 줄인지
```
> 시트는 "링크가 있는 모든 사용자 = 뷰어"로 공개, **헤더 1줄 + 한 제품당 한 줄**인 깨끗한 탭이어야 합니다.

### 2) 필터(facet) 정하기 (`FACETS`)
`hints` 단어가 **헤더 이름에 포함**되면 그 컬럼으로 필터 드롭다운이 자동 생성됩니다.
```js
{ label: "브랜드", hints: ["브랜드", "brand", "제조사"] },
{ label: "사이즈", hints: ["사이즈", "size"], derive: "mm" },   // 폭(mm)만 추출
{ label: "색상",   hints: ["색상", "컬러"],   derive: "color" }, // 표준색 버킷
```
- `derive: "mm"` → 지저분한 규격값에서 폭(20mm 등)만 뽑아 사이즈 필터를 정확하게.
- `derive: "color"` → "미드나잇블루·샌드핑크" 같은 복합 색상명을 표준색(블루·핑크)으로.
- **워치 스트랩이 아니면** `derive`를 빼면 됩니다(원본값 그대로 필터). 색상 버킷을 바꾸려면 `COLOR_BUCKETS` 수정.

### 3) 특수 컬럼 인식 (`COLUMN_HINTS`)
헤더에 해당 단어가 들어가면 자동으로 특별 렌더링됩니다.
| 키 | 용도 |
|---|---|
| `image` | 썸네일 이미지(셀의 이미지URL 또는 `images/` 경로) |
| `link`  | "스토어에서 보기" 버튼(네이버/쿠팡 등 상품 URL) |
| `price` | `12,000원` 포맷 + 우측정렬 + 숫자정렬 |
| `stock` | 재고 배지(품절/부족/정상) + 재고 필터 |
| `name`  | 상세창 제목·카드 제목 |
> 없는 컬럼은 그냥 생략됩니다(가격/재고가 없으면 관련 UI도 자동으로 빠짐).

### 4) 배포
GitHub Pages로 fork → push → Settings ▸ Pages(main/root). 끝.
이미지를 시트 삽입사진에서 뽑아 쓸 경우만 `build.sh`(Python) 파이프라인이 필요하고,
이미 이미지URL 컬럼이 있으면 **파이썬 없이** config만으로 끝납니다.

## GitHub Pages 배포

```bash
git init && git add . && git commit -m "스페이스쉴드 카탈로그"
git remote add origin https://github.com/<아이디>/spaceshied-catalog.git
git push -u origin main
```
Settings → Pages → main / root. `images/`(약 6MB)도 함께 커밋됩니다.

## 파일 구조
```
fetch.py        시트 다운로드 (CSV + xlsx)
normalize.py    변환 + 이미지 추출/리사이즈 → catalog.csv, images/
build_images.py xlsx 삽입이미지 위치(행/열) 파서
build.sh        fetch + normalize 한 번에
config.js       앱 설정 (제목/필터/컬럼 매핑)
index.html, app.js, styles.css   웹앱
catalog.csv     변환 결과 (= sample-data.csv)
images/         추출된 상품 이미지(리사이즈)
```
